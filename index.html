Virtual environments are online platforms where people can work, play, socialize, and build.


Why research virtual environments?

Despite growing demand, existing systems do not support vast, seamless virtual environments.

Scope: performance of metaverses, a category of
virtual environment systems

  • 3D virtual environment with many simultaneous users.
  • Environment accessed through client viewer application.
  • Metaverse content can be built, imported, and modified.
  • User actions are immediately reflected in environment.

Many factors impact virtual environment performance.

  • Unpredictable user behavior
  • Real-time environment state updates
  • Underlying hardware and network limitations
  • Heterogeneous client viewer implementations
RQ-1. How can virtual environments mitigate unexpected behavior from heterogeneous clients?
RQ-2. Which virtual environment content has the highest performance impact?

Contribution of this work

  • Measures impact of an ad hoc performance optimization in OpenSimulator.
  • Systematic study of user login performance impact, previously unmeasured.
  • Motivates the study of automated techniques for virtual environment performance measurement.

Limitations

  • Heavy manual overhead
  • Login study limited to single user
  • Synthetic client bot behavior not comparable to real behavior

Contribution of this work

  • Virtual environment server must mitigate performance; clients not trustworthy.
  • We demonstrate value of offloading heavy asset retrieval during user entrance into a virtual environment.
  • Novel techniques needed to overcome the heavy manual overhead in virtual environment experimentation.

Conclusions

  • Automated techniques needed to address manual overhead of virtual environment performance measurement.
  • Virtual environment servers must mitigate performance issues, rather than relying on client implementations or good user behavior.
